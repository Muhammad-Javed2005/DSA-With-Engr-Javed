# Recursion and Backtracking

Welcome to the **Recursion** module! This section helps you understand functions that call themselves to solve smaller instances of the same problem.

## 🎯 Overview
Recursion is a method of solving a problem where the solution depends on solutions to smaller instances of the same problem. It is fundamental for tree traversals, divide and conquer algorithms, and dynamic programming.

## 📚 What You Will Learn
* **Fundamentals:** Base cases, recursive calls, and the call stack.
* **Patterns:** Tail recursion, multiple recursive calls (Fibonacci, Merge Sort).
* **Backtracking:** Methodically building candidates to solutions and abandoning them ("backtracking") early.
* **Complexity Analysis:** Using Recurrence Relations and the Master Theorem.

## 🚀 Getting Started
1. Always establish a clear "base case" to prevent stack overflow.
2. Draw recursive trees to visualize execution flow.
3. Start with simple mathematical recursion before moving to backtracking on grids/arrays.
